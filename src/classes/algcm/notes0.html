NFA vs. DFA

by: Francheska Vicente
NFA - Non-deterministic Finite Accepters - there can be more than one transition per input per state; there can also be no transition for an input in a certain state (this just means that this input would go to a dead state or outside of the graph, which would result to an immediate termination) DFA - Deterministic Finite Accepters - there should be SPECIFICALLY one transition per input per state (e.g. if {0, 1}*, each state should have transitions for 0 and for 1). NOTES: - For both, - the input is a string - the outputs possible are 1 (if the string is accepted) and 0 (if the string is rejected) - If the two machines are equivalent, then a string that is accepted by the NFA is also accepted by the DFA. - If we want to convert a DFA to an NFA, we don't have to do anything because TECHNICALLY a DFA is an NFA (since it is not required to have more than one transition in a certain state for the same input). I hope this has been helpful in differentiating the two!
